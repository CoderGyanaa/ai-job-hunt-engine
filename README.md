## 📖 What Is This?

**AI Job Hunt Engine** is a single-file, zero-dependency web app that gives every student and professional — from a 1st year college student to a senior engineer — a complete toolkit for finding jobs, internships, and building their career. No account needed. No paid subscription. Runs locally on your computer.

It combines a smart **job link generator** (LinkedIn, Naukri, Indeed, Wellfound, and more) with three powerful **AI features** powered by Google's free Gemini API:

- 🤖 **AI Resume Analyser** — Get expert feedback on your resume in seconds
- 📊 **ATS Score Checker** — Know your match % before you apply
- ✍️ **Cover Letter Generator** — Get a personalised, human-sounding cover letter

> **Built for India, works globally.** All features work with Indian job portals, Indian salary ranges, and Indian hiring patterns — while also supporting global platforms.

## 🖥️ How to Run

> **No installation. No Node. No npm. Just double-click.**

### Option 1 — Python (Recommended, fastest)
```bash
# If you have Python installed:
python -m http.server 8080
# Then open: http://localhost:8080
```
Or just double-click **`START_HERE.bat`** — it auto-detects Python, Node.js, or PHP and launches the server for you.

### Option 2 — VS Code Live Server
1. Open `index.html` in VS Code
2. Install the **Live Server** extension (`Ctrl+Shift+X` → search "Live Server")
3. Click **"Go Live"** at the bottom right

### Option 3 — Direct File (quick test)
Just open `index.html` directly in your browser. Note: PDF parsing works best with a local server.

## ⚙️ Technical Details

### Architecture
```
┌─────────────────────────────────────────────────────┐
│              Single File — index.html               │
│                                                     │
│  User Input → [Hash Cache Check] → sessionStorage   │
│                     ↓ (cache miss only)             │
│            [Input Truncator + Budget Guard]          │
│                     ↓                               │
│         Gemini API  (key in x-goog-api-key header)  │
│         flash-lite (primary) → flash (fallback)     │
│         Exponential backoff: 0s → 8s → 20s          │
│                     ↓                               │
│         Result cached → displayed + copyable        │
└─────────────────────────────────────────────────────┘
```

### AI Optimisations (v8)
| Optimisation | Detail |
|---|---|
| **Response Caching** | Results cached by content hash — same inputs never trigger a second API call |
| **Input Truncation** | Resume capped at 3,000 chars (~750 tokens), JD at 2,000 chars (~500 tokens) |
| **Per-task Temperature** | Analyse: 0.3 · ATS: 0.2 (parser-safe) · Cover: 0.7 (creative) |
| **Proper Output Limits** | Analyse/ATS: 1,800 tokens (full 7 sections) · Cover: 900 tokens |
| **API Key Security** | Key sent via `x-goog-api-key` header, never in the URL |
| **System Instruction** | Uses Gemini's dedicated `system_instruction` field, not merged prompts |
| **Session Call Guard** | Soft limit of 15 API calls/session with friendly UI warning |
| **Model Fallback** | Auto-switches from flash-lite → flash on 429 with countdown |
| **DRY Code** | Single `extractGeminiText()`, `renderResult()`, `renderCached()` helpers |

### Free Tier Limits (Google Gemini)
| Model | Requests/min | Requests/day |
|---|---|---|
| `gemini-2.0-flash-lite` | 30 RPM | 1,500/day |
| `gemini-2.0-flash` | 15 RPM | 1,500/day |

With caching enabled, a typical user session uses **3–5 actual API calls** instead of 10–20.

### No Dependencies, No Build Step
- ✅ Zero npm / yarn / pip
- ✅ Zero backend / server needed
- ✅ PDF parsing via [PDF.js](https://cdnjs.cloudflare.com/ajax/libs/pdf.js/3.11.174/pdf.min.js) (CDN)
- ✅ DOCX parsing via [Mammoth.js](https://cdnjs.cloudflare.com/ajax/libs/mammoth/1.6.0/mammoth.browser.min.js) (CDN)
- ✅ Everything else: vanilla HTML + CSS + JS

## 🛡️ Privacy & Security

| Concern | Answer |
|---|---|
| **Where is my API key stored?** | Only in your browser's `sessionStorage` — cleared when the tab closes |
| **Is my resume uploaded anywhere?** | No. Everything is processed locally in your browser. Your resume never leaves your device. |
| **Does any data go to a server?** | The only external call is to Google's Gemini API, which you're calling with your own key |
| **Is the key visible in network logs?** | No. v8 sends the key via the `x-goog-api-key` HTTP header, not as a URL parameter |
| **Can anyone see my results?** | No. Results are stored only in `sessionStorage` on your own machine |

## 🙋 FAQ

**Q: Do I need to pay for the Gemini API?**  
A: No. Google provides a generous free tier — 30 requests/minute and 1,500 requests/day. For personal use, this is more than enough.

**Q: Does this work on Mac / Linux?**  
A: Yes. `START_HERE.bat` is Windows-only, but you can run `python3 -m http.server 8080` in any terminal and open `http://localhost:8080`.

**Q: My PDF isn't being parsed correctly — what do I do?**  
A: Some scanned PDFs (images, not text) can't be parsed. Try copy-pasting your resume text directly into the text box instead.

**Q: The AI response is getting cut off mid-sentence.**  
A: This was a bug in v7 (maxOutputTokens was too low). v8 uses 1,800 tokens for Analyse and ATS, which handles all 7 sections without cutoff.

**Q: Can I use this without an internet connection?**  
A: The non-AI features (Job Links, Internships, Hackathons, Prepare tabs) work without internet. The AI features require an internet connection to reach the Gemini API.

**Q: Is this safe to use with my real resume?**  
A: Yes. Your resume is processed entirely in your browser and sent only to Google's Gemini API using your own API key. It is never stored on any server.
